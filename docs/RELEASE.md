# Carolus Nexus — Release build

## Quick publish (framework-dependent)

From the repo root or `avalonia/`:

```cmd
cd avalonia
Publish-Release.cmd
```

Output: `avalonia/bin/Release/net10.0-windows/win-x64/publish/`  
The machine must have the **.NET 10** desktop runtime installed.

## Self-contained (no separate runtime)

Larger folder, but runnable on clean Windows installs:

```cmd
cd avalonia
dotnet publish ClippyRW.Avalonia.csproj -c Release -r win-x64 --self-contained true /p:PublishSingleFile=false /p:IncludeNativeLibrariesForSelfExtract=true
```

## Versioning

- Set version in [`avalonia/ClippyRW.Avalonia.csproj`](../avalonia/ClippyRW.Avalonia.csproj) via `<Version>`, `<AssemblyVersion>`, and/or `<FileVersion>` when cutting a tag.
- Secrets stay in `windows/.env` (not in publish output). Copy `windows/.env.example` on target machines.

## Data layout

The app resolves the repo by finding a `windows/` folder upward from the executable. For deployment **outside** the repo tree, ship a layout that still contains `windows/data/` and `windows/.env`, or run from a copy of the repository root.
